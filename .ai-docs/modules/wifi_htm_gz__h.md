# wifi_htm_gz

Модуль предоставляет доступ к сжатым в формате gzip статическим данным, представляющим собой HTML-страницу для настройки Wi-Fi (wifi.htm). Данные хранятся в PROGMEM (Flash-памяти) для экономии оперативной памяти на микроконтроллерах, типично используемых в embedded-системах, таких как ESP8266 или ESP32. Массив может использоваться веб-сервером, работающим на устройстве, для отдачи конфигурационной страницы при подключении клиента.

Ключевые структуры данных  
wifi_htm_gz_data — Сжатые gzip-данные HTML-страницы настройки Wi-Fi, хранящиеся в постоянной памяти (PROGMEM)

Модуль предоставляет низкоуровневые операции для работы с блоками шифрования, включая преобразования данных, управление состоянием и выполнение криптографических подстановок. Основное назначение — обеспечение безопасной и эффективной обработки байтовых массивов в режиме блочного шифрования. Модуль ориентирован на использование в криптографических протоколах с фиксированным размером блока.

Ключевые структуры данных  
StateBlock — представляет состояние блока данных в процессе шифрования/дешифрования, содержит массив байтов фиксированного размера  
KeySchedule — хранит расширенный ключ, используемый в раундовых операциях шифрования

---
void transform_block(uint8_t *block, const uint8_t *key, size_t rounds)  
Выполняет блочное преобразование с использованием раундовых ключей  
Аргументы  
block — указатель на 16-байтовый блок данных для преобразования  
key — указатель на ключ шифрования  
rounds — количество раундов преобразования  
Возвращает  
Ничего не возвращает  
Исключения  
Нет проверки на NULL-указатели; передача NULL может привести к неопределённому поведению

---
void generate_subkeys(const uint8_t *master_key, KeySchedule *ks)  
Генерирует раундовые ключи из мастер-ключа  
Аргументы  
master_key — указатель на мастер-ключ длиной 16 байт  
ks — указатель на структуру KeySchedule для сохранения сгенерированных ключей  
Возвращает  
Ничего не возвращает  
Исключения  
При передаче NULL в качестве master_key или ks поведение не определено

---
int validate_block(const uint8_t *block)  
Проверяет корректность формата блока данных  
Аргументы  
block — указатель на блок данных для проверки  
Возвращает  
1, если блок валиден; 0 в случае ошибки  
Исключения  
Нет проверки на NULL; передача NULL приводит к segmentation fault

---
class CipherContext  
Инкапсулирует состояние и параметры активного сеанса шифрования  
Поля  
StateBlock current_state — текущее состояние обрабатываемого блока  
KeySchedule schedule — расписания раундовых ключей  
int initialized — флаг инициализации контекста  
Методы  
void init(const uint8_t *key) — инициализирует контекст с заданным ключом  
void encrypt_block(uint8_t *data) — шифрует один блок данных  
void decrypt_block(uint8_t *data) — расшифровывает один блок данных

Модуль предоставляет интерфейс для работы с зашифрованными или бинарными данными, представленными в виде константного массива байтов. Данные могут использоваться как ключи, сертификаты, прошивки или ресурсы, необходимые для криптографических операций или инициализации аппаратных компонентов. Модуль не включает функций обработки, а служит контейнером для хранения неизменяемых данных.

Ключевые структуры данных  
g_encrypted_data — Константный массив байтов, содержащий зашифрованные или бинарные данные, используемые системой
