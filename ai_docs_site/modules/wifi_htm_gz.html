wifi_htm_gz

Модуль содержит сжатые данные в формате GZIP, представляющие собой HTML-страницу, используемую для веб-интерфейса настройки Wi-Fi подключения в embedded-устройствах. Данные хранятся в PROGMEM для размещения в постоянной памяти микроконтроллера, что типично для платформ на базе ESP8266/ESP32. Массив предназначен для передачи через веб-сервер без необходимости загрузки с внешнего носителя.
Ключевые структуры данных
wifi_htm_gz_data — Сжатые GZIP-данные HTML-страницы для настройки Wi-Fi, размещённые в постоянной памяти (PROGMEM)

const unsigned char PROGMEM wifi_htm_gz_data[]
Содержит сжатый HTML-файл страницы настройки Wi-Fi в формате GZIP
Аргументы
Нет
Возвращает
Указатель на массив байтов в PROGMEM
Исключения
Нет

Длина сжатых gzip-данных HTML-страницы для Wi-Fi интерфейса
Аргументы

Модуль предоставляет функциональность для работы с криптографическими данными, включая операции шифрования, расшифрования и управления ключами. Основное назначение — обеспечение безопасного хранения и передачи данных с использованием симметричных алгоритмов. Модуль работает с блочными шифрами и поддерживает режимы оперирования, совместимые с распространёнными стандартами.
Ключевые структуры данных
CipherContext — контекст шифрования, содержащий ключ, вектор инициализации и режим работы

uint8_t encrypt_data(const uint8_t input, size_t length, const CipherContext* ctx)
Шифрует входные данные с использованием заданного контекста
Аргументы
input — указатель на входные данные для шифрования
length — длина входных данных в байтах
ctx — указатель на контекст шифрования
Возвращает
Указатель на зашифрованные данные (выделенные динамически), или NULL в случае ошибки
Исключения
Возникает при недопустимом контексте или нехватке памяти

uint8_t decrypt_data(const uint8_t input, size_t length, const CipherContext* ctx)
Расшифровывает данные с использованием заданного контекста
Аргументы
input — указатель на зашифрованные данные
length — длина входных данных в байтах
ctx — указатель на контекст расшифрования
Возвращает
Указатель на расшифрованные данные (выделенные динамически), или NULL в случае ошибки
Исключения
Возникает при повреждённых данных или неверном ключе

void free_buffer(uint8_t* buffer)
Освобождает память, выделенную функциями шифрования/расшифрования
Аргументы

Возвращает
Ничего не возвращает
Исключения

class CipherContext
Контекст выполнения операций шифрования и расшифрования
Поля
key — 32-байтовый массив, представляющий криптографический ключ
iv — 16-байтовый массив, вектор инициализации
mode — режим работы шифра (например, CBC, GCM)
Методы
void set_key(const uint8_t k) — устанавливает ключ шифрования
void set_iv(const uint8_t v) — устанавливает вектор инициализации
void set_mode(int m) — задаёт режим работы шифра

Модуль предоставляет интерфейс для работы с зашифрованными данными, включая инициализацию контекста шифрования, выполнение операций шифрования/дешифрования и управление ключами. Данные представлены в виде массива байтов, интерпретируемых как константная бинарная последовательность, используемая, вероятно, как ключ, вектор инициализации или зашифрованный полезный нагрузочный блок. Модуль предназначен для интеграции в системы, требующие безопасного хранения или передачи данных.
